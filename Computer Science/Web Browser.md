웹 브라우저(Web Browser)는 사용자가 인터넷에서 정보를 검색하고, 웹 페이지를 열고, 이용할 수 있는 소프트웨어입니다. 일반적으로 그래픽 사용자 인터페이스(GUI)를 가지고 있으며, 웹 서버에서 전송된 HTML, CSS, JavaScript, 이미지 등 다양한 형태의 웹 리소스를 받아서 렌더링하여 사용자에게 보여줍니다.

웹 브라우저는 인터넷 익스플로러(Internet Explorer), 구글 크롬(Google Chrome), 모질라 파이어폭스(Mozilla Firefox), 사파리(Safari), 오페라(Opera) 등 여러 가지가 있으며, 각각의 웹 브라우저는 서로 다른 기능과 특징을 가지고 있습니다. 웹 브라우저는 사용자의 요청에 따라 웹 페이지를 다운로드하고 렌더링하여 보여주는데, 이 과정에서 HTML 문서의 구조, CSS 스타일, JavaScript 코드 등을 해석하고 해석 결과물을 사용자에게 보여주는 역할을 합니다.

또한, 최근에는 웹 브라우저가 더 많은 기능을 제공하기 시작했습니다. 예를 들어, 웹 브라우저는 PDF 문서를 보여주거나, 오디오/비디오 파일을 재생할 수 있고, 웹캠과 마이크를 이용하여 실시간 비디오 채팅이나 음성 통화를 할 수도 있습니다. 또한, 웹 브라우저는 쿠키, 캐시, 웹 스토리지 등을 이용하여 사용자의 데이터를 저장하고 관리할 수 있습니다.

## 도메인 이름과 DNS

도메인 이름(Domain Name)은 인터넷 상에서 컴퓨터들을 식별하기 위한 문자열로, 우리가 인터넷 주소창에 입력하는 **[www.example.com과](http://www.example.xn--com-rg8l/)** 같은 형식으로 표현됩니다. 이는 IP 주소를 기억하기 어려운 사람들이 이해하기 쉬운 형태로 변환된 것입니다.

DNS(Domain Name System)은 인터넷에서 도메인 이름을 IP 주소로 변환해주는 시스템입니다. DNS 서버는 전 세계적으로 분산되어 있으며, 사용자가 입력한 도메인 이름을 해석하여 해당 도메인 이름에 해당하는 IP 주소를 반환해줍니다. 이를 통해 사용자는 도메인 이름을 통해 해당 서버에 접속할 수 있습니다.

예를 들어, 사용자가 웹 브라우저에서 **[www.example.com에](http://www.example.xn--com-568n/)** 접속하려고 할 때, 먼저 웹 브라우저는 DNS 서버에 **[www.example.com의](http://www.example.xn--com-yh0o/)** IP 주소를 요청합니다. DNS 서버는 **[www.example.com에](http://www.example.xn--com-568n/)** 해당하는 IP 주소를 반환해주고, 웹 브라우저는 해당 IP 주소로 서버에 접속하여 웹 페이지를 불러옵니다. 이러한 과정을 통해 도메인 이름을 통해 서버에 접속할 수 있게 됩니다.

## 웹 렌더링

**웹 렌더링(Web Rendering)**
은 서버로부터 받은 리소스를 이용자에게 시각화하는 행위를 말합니다. 서버의 응답을 받은 웹 브라우저는 리소스의 타입을 확인하고, 적절한 방식으로 이용자에게 전달합니다. 예를 들어, 서버로부터 HTML과 CSS를 받으면 브라우저는 HTML을 파싱하고 CSS를 적용하여 이용자에게 보여줍니다.
